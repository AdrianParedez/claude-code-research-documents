Comparison artifact

Claude prompting, Claude Code prompting, and Claude Code systems are not the same object.

If those three papers get flattened into one generic prompt-engineering conversation, the actual control surface disappears. This page keeps the distinction compact.

Executive point

Claude prompting optimizes a task contract. Claude Code prompting optimizes a governed agent brief inside a coding loop. Claude Code systems analysis evaluates the operating stack that makes that loop possible.

01
Object / Model-task contract Claude prompting

Prompt the model, not the whole operating environment.

The main job is defining the task, output contract, structure, and evaluation target tightly enough that Claude does not have to guess what success means.

Optimize
Output quality for a defined task
Control surface
Spec, XML, examples, and evals

02
Object / Agent brief Claude Code prompting

Prompt the agent inside the repo loop.

The main job is defining outcome, evidence, autonomy, constraints, verification, and handoff while leaving the agent room to inspect, choose tools, and prove completion.

Optimize
Bounded coding-task completion
Control surface
Brief, context load, permissions, and proof of done
Read first
MS-02 manuscript
03
Object / Operating stack Claude Code systems

Evaluate the stack, not only the live prompt.

The main job is deciding what Claude Code has become as a system: tool harness, context manager, permissions layer, memory layer, extensibility surface, and deployment model.

Optimize
Architecture, governance, and operating fit
Control surface
Tools, memory, skills, permissions, and surfaces

Matrix

Different layer, different benchmark.

Each paper answers a different bounded question. That is why the repo should not be read as one merged prompt-engineering document.

Question Claude prompting Claude Code prompting Claude Code systems
Object under analysis Claude as a model family responding to a defined task Claude Code as a coding agent receiving a live work brief Claude Code as a product and operating environment
Main question What prompt architecture most reliably improves task performance for Claude? What prompt architecture most reliably improves bounded coding-task completion in Claude Code? What should Claude Code be understood as: wrapper, tool, or governed agent system?
Main control surface Task definition, output contract, XML structure, examples, and evals Outcome, evidence, autonomy, constraints, verification, and handoff Tools, permissions, context management, memory, skills, subagents, surfaces, and deployment model
Common mistake Treating good prompts like magic phrasing Treating the environment like a one-shot chat box Treating Claude Code like a thin CLI around chat
Primary output Task-contract framework plus Claude templates Governed agent-brief framework plus Claude Code templates Systems frame for evaluation, rollout, and governance
